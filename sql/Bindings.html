2.1.5 Bindings

Many syntactic forms in Pyret need to designate names for values. These are uniformly represented as ‹binding›s:

‹binding›: [shadow] NAME [:: ‹ann›] | ‹tuple-binding› ‹tuple-binding›: { ‹tuple-bind-list› } ‹tuple-bind-list›: (‹bind-tuple›)* ‹binding› [;] ‹bind-tuple›: ‹binding› ;

One form of binding is ‹tuple-binding›s. When a tuple is bound using a tuple binding each value in the tuple is assigned to its corresponding identifier.

Nested binding will be added in the future.

Nested tuples cannot be bound using ‹tuple-binding›s. The number of indentifiers must match the length of the given tuple.

Examples:
check:
  {x;y} = {1;2}
  x is 1
  y is 2

  fun sum-two({k;v;}, {a;b;c;}):
    k + v + a + b + c
  end

  sum-two({10; 12}, {1; 4; 5}) is 32

  fun sum-vals(elts) block:
    var sum = 0
    for each({k;v;} from elts):
      sum := sum + v
    end
    sum
  end

  elts = [list: {"a"; 5}, {"b"; 6}, {"c"; 7}]
  sum-vals(elts) is 18
end
